Software Development Framework Requirements Software and Hardware Feasibility Issues

3.1 Software Development Framework

The proposed CLOIE system follows an Agile Kanban-based workflow to support iterative delivery and continuous refinement in a capstone environment.

  • Backlog: planned features, requirements, and improvements
  • To Do: prioritized tasks ready for implementation
  • In Progress: active development
  • Testing: validation of functionality and usability
  • Done: verified and completed tasks

3.2 Requirements

3.2.1 User Requirements

Stakeholder groups include faculty, students, graduating students, alumni, industry partners, program heads/deans, and system administrators.

3.2.2 Functional Requirements

Core modules include user and access management, outcomes management, instrument management, evaluation cycles, response processing, analytics/reporting, and administrative controls.

3.2.3 Non-Functional Requirements

Priority quality attributes include usability, performance, security, confidentiality, reliability, scalability, availability, responsiveness, and maintainability.

3.2.4 Software and Hardware Requirements

Note: this section summarizes the planned technology stack of the proposed CLOIE system in the capstone documentation, not the implementation stack of this summary website.

Software Requirements

  • Frontend framework: Next.js
  • Language: TypeScript
  • Styling and UI: Tailwind CSS and shadcn/ui
  • Forms/validation/state: React Hook Form, Zod, TanStack Query
  • Data and deployment: PostgreSQL, Supabase, Vercel
  • Visualization/testing: Chart.js, Recharts, wordcloud2.js, Vitest, Playwright

Hardware Requirements

  • Development: modern laptop/desktop, at least 8 GB RAM, SSD storage
  • End users: desktop/laptop/tablet/smartphone with modern browser and internet

3.3 Feasibility Issues

Economic Feasibility

Web/PWA-based development is practical for student-led implementation and avoids costs from multi-platform native builds.

Technical Feasibility

The required modules are compatible with modern web architecture and can be delivered incrementally.

Operational Feasibility

The proposed workflows align with existing academic evaluation practices at Assumption College of Davao.

Schedule Feasibility

The staged capstone timeline (March to September 2026) supports progressive delivery from core features to refinement.
